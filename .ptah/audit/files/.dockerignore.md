# File: .dockerignore

Keeps the `tsk-002` build context to what the frozen runtime needs. Excludes
`.git`, `.ptah`, `nbproject`, `user_guide`, `vendor`, `composer.lock`, `*.log`,
`ptah-debug.log`, `.DS_Store`, and `.env` (the git-ignored file
`docker-compose.yml` sources `MYSQL_ROOT_PASSWORD` from — see
`legacy_debt.md` SEC-2 and `files/docker-compose.yml.md` — so the live
credential never bakes into a build-context layer either). `system/` (the CI
framework) is **not** excluded — it is required at runtime.

## Blast radius

New file; build-context scoping only. Excluding `.ptah` means the KB itself
never enters the image; excluding `vendor`/`composer.lock` means the image's
own `composer install` step (see `files/Dockerfile.md`) always resolves
dependencies fresh rather than trusting a stale host `vendor/`.
